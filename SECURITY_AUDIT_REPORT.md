**Date:** April 21, 2026  
**Target:** Vanguard Cross-Platform Messenger (Web, Android, Windows)

## 1. Cryptographic Standards
The system has been unified under a "Maximum Secrecy" policy, moving away from legacy RSA and lower-bit curves.

*   **Asymmetric Encryption:** ECDH (Elliptic Curve Diffie-Hellman) using the **P-521** curve (the strongest NIST curve, offering 256 bits of security strength).
*   **Signatures:** ECDSA with **SHA-512** for authorship verification and identity binding.
*   **Key Agreement:** **X3DH** (Extended Triple Diffie-Hellman) for asynchronous session initiation.
*   **Symmetric Encryption:** **AES-256-GCM** for all data-at-rest and in-transit (Double Ratchet prepared).
*   **KDF:** **PBKDF2** with **SHA-512** and 600,000 iterations (OWASP 2024 compliance) for master key derivation.

## 2. Infrastructure Hardening
### Web (React/Vite)
*   **Encrypted Storage:** Private keys in IndexedDB are encrypted using a password-derived master key. Raw keys are NEVER stored on disk.
*   **Social Verification:** Posts are signed client-side. The feed verifies signatures using the author's public key (fetched via user metadata join).
*   **Device Binding:** Implemented fingerprinting and ticket-based WebSocket authentication.

### Android (Kotlin/JVM)
*   **Engine Migration:** Obsolete RSA code replaced with native `Security` provider ECDH P-521.
*   **Secure API:** WebSocket client updated to handle one-time authentication tickets.

### Windows (Tauri/Rust)
*   **Rust Bridge:** Implemented a secure native hook for sensitive key storage.
*   **Build Integrity:** Restored `Cargo.toml` and `tauri.conf.json` with strict CSP and scope isolation.

### Backend (Go/Gin/PostgreSQL)
*   **Pre-key Vault:** Implemented `/api/crypto/prekeys` for batch OTPK (One-Time Pre-keys) management.
*   **Session Security:** WebSocket tickets prevent hijacking and replay attacks.
*   **Scalability:** Prepared repository patterns for Google Spanner/Bigtable transition.
